Black is an opinionated Python code formatter. It reformats your code to follow a consistent style, making collaboration and code review easier.

üí° Basic Usage

$ pip install black
$ black your_file.py
$ black . # Format all files in the current directory

üîç Check Formatting Without Changing Files

$ black --check .
Tip: black --check . returns a nonzero exit code if any files would be reformatted. Useful for CI!
$ black --diff your_file.py
$ black --diff .
Note: black --check only tells you if files would be reformatted, but does not show a diff.
Use --diff to see what would change.

üìù Example: Before and After

Before: 
def foo(   x,y): print(  "Hello"  )
After running black: 
def foo(x, y):
    print("Hello")

ü§ñ Automate with Pre-commit


-   repo: https://github.com/psf/black
    rev: 24.3.0
    hooks:
      - id: black
        language_version: python3
        args: [--check]
$ pre-commit install
$ pre-commit run --all-files

üñ§ VS Code Integration

  • Install the official Python extension for VS Code.
  • In your settings.json, set:
{
  "python.formatting.provider": "black",
  "editor.formatOnSave": true
}
  • Now, every time you save a Python file, black will automatically format it.
  • You can also run Format Document (Shift+Alt+F) manually.
Tip: black is opinionated and will overwrite your formatting to match its style. Use it to keep your codebase consistent and avoid style debates.
